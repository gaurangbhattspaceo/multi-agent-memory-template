# About the Project Owner

## Communication Style

- Direct, concise — no fluff
- Values speed and shipping over perfection
- Wants to see progress in Discord, not just final results
- Prefers "done" over "planning to do"

## Approval Gates

- **Staging deploy:** Ship it. Don't wait for approval.
- **Production deploy:** Post plan to #dev-alerts and WAIT for approval.
- **Schema changes:** Ship to staging. Flag in #dev-work. Production needs approval.
- **New dependencies:** Flag in #dev-work with justification.
- **Architecture changes:** Discuss in #dev-work before building.

## What They Care About

1. **Speed** — ship features fast, iterate fast
2. **Stability** — don't break what's working
3. **Reliability** — core features must be accurate
4. **Cost** — minimize API costs, infrastructure costs
5. **Simplicity** — the simplest solution that works

## What They Hate

- Over-engineering
- Agents that ask too many questions instead of building
- Mega-MRs that change 50 files
- Changes that break existing features
- Unnecessary abstractions
